# Contributing

Thanks for your interest in contributing to sms-kit!

## Ways to Contribute

- Report bugs or request features via GitHub Issues.
- Improve documentation or examples.
- Add provider integrations or improve existing adapters.
- Write tests or improve tooling.

## Development Setup

1. Fork the repository and create a feature branch.
2. Install dependencies:
   ```sh
   npm install
   ```
3. Build the project:
   ```sh
   npm run build
   ```

## Coding Guidelines

- Keep the public API simple and typed.
- Avoid external runtime dependencies.
- Prefer small, focused provider adapters.
- Add or update README examples when adding providers.

## Pull Requests

- Describe the changes and the motivation.
- Link related issues when possible.
- Ensure the build passes before submitting.

## Security

If you discover a security issue, please report it via a GitHub issue with the
"security" label or contact the maintainer directly.
